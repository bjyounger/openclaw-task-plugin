## 🔧 修复的问题清单

### 1. destroy() 方法事件触发顺序问题

**严重程度**: 中等  
**发现时间**: Day 3 集成测试  
**影响范围**: 所有依赖 `manager:destroyed` 事件的代码

**问题描述**:
```typescript
// 原来的实现
async destroy(): Promise<void> {
  // ...
  this.eventEmitter.clearAll(); // 先清空监听器
  this.destroyed = true;
  this.emit('manager:destroyed', {...}); // 再触发事件 - 无法接收到
}
```

**修复方案**:
```typescript
// 修复后的实现
async destroy(): Promise<void> {
  // ...
  this.destroyed = true;
  this.emit('manager:destroyed', {...}); // 先触发事件
  this.eventEmitter.clearAll(); // 再清空监听器
}
```

**验证**: ✅ 测试通过
